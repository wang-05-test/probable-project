# probable-project
软件测试学习与作业仓库
## 项目简介
本仓库用于系统记录软件测试全流程学习进度，涵盖功能测试、用例设计、缺陷管理、自动化测试等核心模块，同步沉淀学习笔记、测试用例、思维导图、缺陷报告等实战资料，打造标准化学习作品集，为软件测试岗位求职做准备。
## 📂 仓库结构
probable-project/
01 - 功能测试基础 / # 软件测试基础理论学习
（1）用例 / # 功能测试用例文件
（2）脚本 / # 自动化测试脚本（后续学习）
（3）报告 / # 测试总结报告
（4）笔记 / # 学习笔记、文档
（5）截图 / # 思维导图、实操截图
02 - 用例设计 / # 用例设计方法学习（等价类、边界值等）
03 - 缺陷管理 / # 缺陷提交、跟踪、管理相关学习
04 - 综合实战 / # 项目实战、综合练习
README.md # 仓库说明文档 ── .gitignore # Git 忽略文件配置
## 使用说明
可直接下载  .emmx  文件，使用 XMind 或兼容思维导图工具打开查看。
后续将逐步补充测试用例、缺陷报告、自动化测试脚本等内容。
## 学习目标
1.  系统掌握软件测试全流程，能独立完成电商等项目的功能测试
2.  熟练掌握用例设计方法，输出规范的测试用例与缺陷报告
3.  掌握Python自动化测试技术，具备自动化测试能力
4.  打造标准化GitHub作品集，顺利拿到软件测试岗位offer
